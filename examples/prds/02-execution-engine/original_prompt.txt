Build a trade execution engine that safely interfaces between LLM trading decisions and cryptocurrency exchange APIs.

CORE COMPONENTS:

1. Order Management System
   - Support order types: market, limit, stop-loss, take-profit, trailing stop
   - Order lifecycle management: pending → submitted → partial → filled/cancelled
   - CCXT library integration for unified exchange API
   - Order queuing with priority levels
   - Idempotency keys to prevent duplicate orders

2. Position Tracking & P&L
   - Real-time position monitoring across all exchanges
   - Mark-to-market P&L calculation
   - Realized vs unrealized gains tracking
   - Average entry price and cost basis
   - Position history with full audit trail

3. Risk Management Layer (CRITICAL)
   - Max position size limits per asset (e.g., max 10% of portfolio in single asset)
   - Portfolio-level exposure caps (e.g., max 50% total crypto exposure)
   - Drawdown circuit breakers: pause trading if daily loss > 5%
   - Cooldown periods after consecutive losses
   - Correlation limits: prevent overexposure to correlated assets
   - Leverage restrictions: max 2x leverage, no margin calls

4. LLM Output Validation
   - Use Guardrails AI to enforce valid JSON trade signals
   - Schema validation: action (BUY/SELL/HOLD), symbol, quantity, confidence
   - Reject malformed or incomplete signals
   - Sanity checks: reject impossible quantities, invalid symbols

5. Safety Override Logic
   - Block dangerous suggestions: 100% allocation, unknown assets
   - Rate limiting: max N trades per hour
   - Manual approval required for trades above threshold
   - Kill switch: emergency halt all trading
   - Whitelist of allowed trading pairs

6. Paper Trading Mode
   - Simulated execution with real market data
   - Virtual portfolio tracking
   - Slippage simulation based on order size
   - Performance comparison vs live trading
   - Seamless switch between paper and live modes

7. Multi-Exchange Support
   - Unified interface across Binance, Coinbase, Kraken
   - Automatic failover if primary exchange is down
   - Best execution: route to exchange with best price
   - Balance synchronization across exchanges

EXECUTION REQUIREMENTS:
- Execute validated LLM decisions within 100ms
- Hard risk constraints cannot be overridden by LLM
- Full audit log of all trade decisions and executions
- Graceful degradation: continue safely if components fail

TECH STACK PREFERENCES:
- Python with FastAPI for REST endpoints
- CCXT for exchange connectivity
- PostgreSQL for order/position persistence
- Redis for real-time state and rate limiting
