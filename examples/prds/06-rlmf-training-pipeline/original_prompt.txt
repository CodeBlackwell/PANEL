Build a training pipeline to fine-tune LLMs using Reinforcement Learning from Market Feedback (RLMF), inspired by the FinRLlama research paper.

PIPELINE COMPONENTS:

1. Historical Trade Labeling System
   - Input: Historical LLM trading decisions with outcomes
   - Labeling criteria:
     * Profit/loss amount and percentage
     * Risk-adjusted return (contribution to Sharpe ratio)
     * Maximum adverse excursion (worst drawdown during trade)
     * Trade duration vs expected holding period
   - Output: Labeled dataset with quality scores (-1 to +1)
   - Format: (prompt, response, reward_score) triplets
   - Storage: Parquet files for efficient training data loading

2. Reward Model
   - Architecture: Fine-tuned LLM or separate regression model
   - Inputs: LLM trading decision (action, reasoning, confidence)
   - Predicts: Expected risk-adjusted return score
   - Training data: Labeled historical trades
   - Loss function: MSE between predicted and actual outcomes
   - Regularization: Penalize overconfident wrong predictions

3. PPO/DPO Training Loop
   - Base model: Llama-3-8B or Mistral-7B (open-source)
   - Fine-tuning approach: LoRA for parameter efficiency
   - PPO (Proximal Policy Optimization):
     * Actor: LLM generates trading decisions
     * Critic: Reward model evaluates decisions
     * Update policy to maximize expected reward
   - DPO (Direct Preference Optimization) alternative:
     * Pairs of (better, worse) trading decisions
     * Train LLM to prefer higher-reward decisions
     * Simpler than PPO, no separate reward model needed
   - Hyperparameters: learning rate, KL penalty, batch size

4. Evaluation Framework
   - Held-out test periods (e.g., last 3 months of data)
   - Metrics:
     * Sharpe ratio improvement vs baseline
     * Win rate change
     * Average profit per trade
     * Maximum drawdown comparison
     * Calibration: does confidence match accuracy?
   - Statistical significance testing (t-test, bootstrap)
   - Ablation studies: which training signals help most

5. A/B Testing Infrastructure
   - Traffic splitting: X% to new model, (100-X)% to baseline
   - Gradual rollout: 5% → 20% → 50% → 100%
   - Real-time metric comparison dashboard
   - Automatic rollback if new model underperforms
   - Shadow mode: new model predicts but doesn't execute

TRAINING REQUIREMENTS:

- Support consumer GPUs: RTX 3090/4090 (24GB VRAM)
- Quantization: 4-bit or 8-bit for memory efficiency
- Gradient checkpointing for long context
- Mixed precision training (FP16/BF16)
- Distributed training support for multiple GPUs

DATA REQUIREMENTS:

- Minimum 1000 labeled trades for initial training
- Continuous learning: add new trades weekly
- Data augmentation: synthetic variations of real trades
- Handle class imbalance (more HOLD than BUY/SELL)

EXPECTED OUTCOMES:

- LLM learns to avoid high-risk trades
- More consistent confidence calibration
- Tighter, more profitable trading decisions
- Reduced drawdowns during volatile periods

TECH STACK:
- Hugging Face Transformers for model loading
- PEFT library for LoRA fine-tuning
- TRL library for RLHF/DPO training
- Weights & Biases for experiment tracking
- PyTorch with CUDA for training
- Optionally: DeepSpeed for memory optimization
